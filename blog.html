Q.1: Discuss the scope of var, let, and const?

Answer: var is Function-scoped, hoisted to the top of the function and can be redeclared and reassioned.
let is block scoped, hoisted to the top of the block and can be reassigned but not redeclared in the same scope.
const is block-scoped, hoisted to the top of the block and cannot be reassigned after initial assignment, but properties of objects and elements of arrays can be modified.

Q.2: Tell us the use cases of null and undefined.

Answer: undefined typically represents the absence or uninitialized state of a variable, often used when something hasn't been defined or assigned.
null is used to intentionally indicate the absence of a value, often used when you want to clear a variable or property.

In summary, undefined usually means something wasn't defined or initialized, while null signifies a deliberate absence of a value.

Q.3: What do you mean by REST API?

Answer: REST API is a set of rules and principles for building web services that allow different software applications to communicate with each other over the internet. It uses standard HTTP methods and URLs to perform operations on data, making it a widely used and scalable approach for designing APIs. REST APIs are known for their simplicity, statelessness, and flexibility in handling various types of data.
